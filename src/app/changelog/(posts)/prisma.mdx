Se integró Prisma ORM a la aplicación entera.

## Antecedentes

De un inicio se utilizó el servicio de Supabase que nos proporciona una base de datos, una API REST para acceder a ella, un storage para archivos y otros servicios mas. Sin embargo, la biblioteca de supabase no es tan completo como un ORM. Además, el modelo de datos de la aplicación es complejo y se necesita una herramienta que nos ayude a modelar y acceder a la base de datos, ya que el modelado en SQL constantemente se modificaba y mantenerse al dia con los cambios entre los distintos entornos de desarrollo do los distintos developers y el de pruebas y produccion desplegados.

## Decicion

Se decidió cambiar a Prisma ORM por las siguientes razones:
- El servicio de Supabase utiliza una base de datos PostgreSQL, la cual es soportada por Prisma.
- El modelado de datos es mas sencillo y se puede hacer en un lenguaje mas sencillo y entendible.
- Prisma ORM es una herramienta mas completa y con mas funcionalidades que el servicio de Supabase.
- Las queries son mas sencillas de hacer y mas entendibles, ademas de que se pueden hacer queries mas complejas que con Subapase.
- El sistema de migraciones mantienen la base de datos actualizada con el modelo de datos de la aplicación.
- Su sistema de typing es muy completo y nos ayuda a evitar errores en tiempo de building y ejecución.
